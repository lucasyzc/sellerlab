# Data EDE Logo System

## Overview
This repository uses **Data EDE** as the primary website brand and **SellerLab** as the product suite brand.

The logo system is built around a compact **D + E monogram** that communicates:
- `Data`: stable infrastructure and clear inputs
- `Engine`: motion, transformation, and decision support
- `Growth`: a directional data path that suggests momentum without using generic ecommerce clichés

## Visual Direction
- Tone: professional, analytical, global, precise
- Brand feel: light-tech business, not consumer-gadget or startup-gimmick
- Shape language: rounded geometry for trust and clarity
- Accent language: cyan data-flow details over a deep navy base

## Asset Set
- `public/logo.svg`
  Square brand mark for UI placements such as headers, footers, and cards.
- `app/icon.svg`
  App-router icon/favicon source for browser tabs and pinned icons.
- `public/logo-lockup.svg`
  Horizontal master logo for presentations, docs, and future hero placements.
- `public/logo-sellerlab-lockup.svg`
  Sub-brand lockup for product-suite contexts where `SellerLab by Data EDE` should appear together.

## Palette
- Base navy: `#081a33`
- Mid navy: `#0d2e58`
- Teal accent: `#126f8b`
- Cyan highlight: `#4fd1e8`
- Light cyan: `#d8fbff`
- Warm white mark: `#effbff`

## Usage Notes
- Prefer the square mark in tight UI spaces and at favicon sizes.
- Use the horizontal lockup when the brand needs to read immediately as `Data EDE`.
- Keep `SellerLab` as a secondary brand expression, not the master site identity.
- Do not add marketplace icons, shopping carts, rockets, or globes to the master logo.
- Preserve clear space around the icon equal to at least one quarter of the mark width.
