# MVP 功能范围（冻结版 v1）

## 1) 目标

用最小成本验证这件事：  
“跨境卖家会不会愿意使用工具并留下联系方式”。

## 2) 必做功能（P0）

- `Landing 页面`
  - 价值说明（效率、上手快）
  - 核心 CTA：立即使用工具
- `工具页（首个工具）`
  - 建议先做：利润计算器
  - 支持输入成本、售价、运费、平台费，输出利润与利润率
- `留资表单`
  - 字段：邮箱（必填）、店铺阶段（可选）
  - 提交成功后展示“下载模板/预约试用”二次 CTA
- `基础事件采集`
  - 页面访问、工具使用、CTA 点击、表单提交成功

## 3) 暂不做（P2/P3）

- 多账号体系与团队权限
- ERP 连接与一键执行
- 复杂自动化规则引擎
- 多租户报表系统

## 4) 成功标准（首月）

- 至少 1 个工具稳定可用
- 每周可输出漏斗：访客 -> 工具使用 -> 留资
- 有第一版激活定义与周报机制

## 5) 技术取舍建议（为了快）

- 前端：Next.js 或纯静态站均可
- 后端：Serverless 函数处理表单
- 数据：先用 CSV/Google Sheet，不强制 DB
- 分析：GA4 或 Plausible 二选一

## 6) 风险与应对

- 风险：做太多导致上线慢  
  对策：严格只交付 P0
- 风险：没有数据闭环  
  对策：先保证 4 个关键事件可导出
- 风险：转化低  
  对策：优先优化工具结果页 CTA 文案和位置
