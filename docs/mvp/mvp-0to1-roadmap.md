# 工具站 MVP 起步包（0->1）

适用前提：当前没有现成数据库、没有 ERP 系统、没有工具站代码基础。  
目标：在 2-4 周内上线一个可用的工具站 MVP，先验证“流量 -> 留资 -> 激活”闭环。

## 一、2-4 周落地节奏

### 第 1 周：确定边界 + 设计可发布版本

- 明确只做 1 个工具（建议：利润计算器）
- 明确 1 个转化动作（邮箱留资）
- 产出页面文案、字段、埋点字典
- 建立手工周报（CSV/Sheet）

验收标准：

- 有一份冻结版范围文档（做什么/不做什么）
- 有可填写的周报模板并首次填报

### 第 2 周：完成 MVP 开发

- 完成 3 个页面：Landing、Tool、Lead Form
- 完成核心流程：访问 -> 使用工具 -> 提交邮箱
- 事件先落本地文件或简易日志（不依赖数据库）

验收标准：

- 本地可跑通全流程
- 能导出周报所需原始事件

### 第 3 周：上线与首轮数据验证

- 部署静态站 + 轻后端（可选 serverless）
- 接入基础统计（GA4 或 Plausible 二选一）
- 开始每周经营复盘（手工）

验收标准：

- 有真实访客数据
- 有第一周漏斗数据（访客、工具使用、留资）

### 第 4 周（可选）：优化和补齐

- 增加第 2 个高意图工具
- 优化 CTA 与表单转化
- 评估是否进入 Postgres 自动化阶段

## 二、当前阶段不做（避免分散）

- 不做 ERP 集成
- 不做 SSO
- 不做复杂权限系统
- 不做全量 BI 平台

## 四、是否进入“数据库自动化”阶段的门槛

满足以下任意 2 条再上 Postgres：

- 连续 4 周有稳定访客（例如每周 > 500 UV）
- 每周留资样本量足够（例如每周 > 30）
- 需要按渠道/语言/工具维度自动切片
- 手工周报耗时超过 2 小时/周
