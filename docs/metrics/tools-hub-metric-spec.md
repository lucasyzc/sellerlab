# 工具站阶段指标口径（北极星 + 周度经营看板）

## 1. 目标与适用范围

本口径用于“工具站独立阶段（Phase 1）”的统一经营度量，覆盖：

- 北极星指标：每周新增激活账号
- 双辅指标：SEO 与商业转化
- 稳定性护栏：工具服务和关键链路健康

所有团队（产品、运营、增长、技术、销售）使用同一份定义，避免口径不一致。

## 2. 北极星指标定义

### 2.1 指标名称

- 中文：每周新增激活账号
- 英文：Weekly New Activated Accounts (WNAA)

### 2.2 指标定义

在统计周内完成注册，且在注册后 7 天内完成至少 1 个核心动作的账号数。

核心动作（任一满足即激活）：

- 同步店铺（`StoreSynced`）
- 刊登发布（`ListingPublished`）
- 发货出库（`ShipmentDispatched`）
- 处理消息（`MessageProcessed`）

### 2.3 统计口径

- 统计对象：`account_id`（去重后）
- 时间粒度：周（自然周，周一 00:00:00 到周日 23:59:59，UTC+8）
- 注册判定事件：`AccountRegistered`
- 激活窗口：`event_time <= registered_at + 7 days`
- 去重规则：
  - 同一账号同周重复触发核心动作仅计 1 次激活
  - 一个账号只在其“注册所在周”的新增激活中计入 1 次
- 排除范围：
  - 内部测试账号（`is_internal = true`）
  - 明显机器人或异常流量账号（反作弊标签）

### 2.4 公式

`每周新增激活账号 = count(distinct account_id where 注册在本周 and 7日内完成>=1核心动作)`

并行跟踪拆解指标：

- `7日激活率 = 每周新增激活账号 / 每周新增注册账号`
- `激活中位时长 = median(首次核心动作时间 - 注册时间)`

## 3. 事件与字段规范（最小集）

所有事件建议统一字段：

- `event_name`：事件名
- `event_time`：事件发生时间（ISO8601）
- `account_id`：账号 ID
- `user_id`：用户 ID（可空）
- `channel`：来源渠道（seo / paid / direct / referral / other）
- `locale`：语言（zh-CN / en-US）
- `tool_id`：工具标识（工具相关事件必填）
- `is_internal`：是否内部账号

关键事件：

- `AccountRegistered`
- `ToolUsed`
- `CtaClicked`
- `EmailCaptured`
- `TrialApplied`
- `StoreSynced`
- `ListingPublished`
- `ShipmentDispatched`
- `MessageProcessed`

## 4. 周度经营看板（核心视图）

建议按“总览 + 漏斗 + 分层 + 稳定性”四个区块搭建。

### 4.1 总览区（必须）

- 每周新增激活账号（WNAA）及 WoW
- 每周新增注册账号及 WoW
- 7日激活率
- 激活中位时长（小时）

### 4.2 漏斗区（必须）

漏斗链路：

`访客 -> 工具使用 -> 注册 -> 店铺授权 -> 首次刊登/发货/消息处理 -> 激活`

每层显示：

- 人数（去重）
- 转化率（相邻步骤）
- 环比变化（WoW）

### 4.3 SEO 与商业双辅指标（建议）

SEO：

- 自然访客 UV
- 工具页转化率（工具使用 / 工具页 UV）
- Top 20 关键词平均排名

商业：

- MQL 数
- 试用申请数与试用转化率
- 试用到付费转化率
- 月流失率（周看板展示 rolling 30d）

### 4.4 稳定性护栏（建议）

- 工具服务可用性（SLO）
- API 成功率
- 关键任务失败率（例如工具结果页生成失败）
- P1/P2 事件数

## 5. 经营看板周节奏（运营机制）

- 周一：刷新上周数据、输出异常波动清单
- 周二：跨团队复盘（增长/运营/产品/技术）
- 周三到周五：针对异常项执行实验与修复
- 下周一：回看实验结果并沉淀 SOP

固定复盘问题：

- 新增激活账号变化来自“注册端”还是“激活端”？
- 渠道结构是否变化（SEO / 非SEO）？
- 哪个核心动作成为激活瓶颈？
- 稳定性问题是否影响转化？

## 6. 数据质量与治理

- 指标字典统一维护，修改需记录版本
- 事件埋点变更先灰度，确保可回滚
- 关键指标设置阈值告警（例如 7日激活率低于目标线）
- 每周检查缺失字段率（`account_id`、`event_time`、`channel`）

## 7. 建议目标（Phase 1 可调整）

- 每周新增激活账号：持续 WoW 正增长
- 7日激活率：先达到并稳定在 20%~30% 区间
- 激活中位时长：逐步压缩到 24 小时内
- 工具服务可用性：>= 99.9%
